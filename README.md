Под графиками функций цветом нарисовать плотность распределения данных (по соответствующему признаку), см. пример в NAM (Neural Additive Models).

## Task 5
Требуется изменить существующий код минимальным образом, чтобы сделать интерпретацию модели более корректной. Для этого надо добавить токен <MASK>, обучить модель с ним, маскируя токены входа (и возможно выхода) случайным образом. Затем поменять NumeralExplainerWrapper, чтобы предсказания модели можно было получить по фрагментам предложения (в коде присутствие токена определяется маской `coalitions`).
В результате объяснение последней цифры должно затрагивать только младшие разряды, например: в числе 1234 = MCCXXXIV на токен предсказания V должна влиять в основном цифра 4.
Задание со 🌟: по полной последовательности (например `<SOS>11<SEP>XI<EOS><PAD>`) строить объяснение насколько хорошо модель предсказывает данный текст (т.е. рассматривать как выход всей модели не вероятность токена, а perplexity). Попробовать объяснить корректную и некорректную последовательности; для некорректной последовательности объяснение должно указывать на позицию ошибки.
